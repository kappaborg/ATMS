Project Assignment
Project Scenario:
Each group of 5 students will be assigned a specific project scenario from the list of 25 topics provided
(e.g., Online Food Delivery Platform, Library Management System, Fitness Tracking App, etc.). Based
on the assigned project, students will work in group to identify , document and present all relevant
software requirements and develop corresponding use cases
Instructions:
1. Understand the Scenario:
o Carefully read and understand the project scenario you are assigned.
o Identify the key actors, user roles, and primary functionalities required by the 
system.
2. Elicit Use Cases:
o Identify at least 3 primary use cases based on the system scenario.
o For each use case, develop a detailed Use Case Description using the Use Case 
Template (include preconditions, postconditions, main flow, alternate flows, and 
exceptions).
3. Create a Use-Case Diagram:
o Using UML or any other modelling tool, create a Use-Case Diagram for your project, 
showing the relationships between actors and use cases.
4. Document Functional and Non-Functional Requirements:
o Use the Software Requirements Specification (SRS) Template to document at least 
10 functional requirements and 5 non-functional requirements for the system.
5. Develop a Requirements Traceability Matrix (RTM):
o Use the RTM Template to link each requirement to its corresponding use case and 
ensure all requirements are traceable.
6. Change Request Management:
o Create a Change Request Form to simulate a change to one of your requirements. 
Document the reason for the change and perform an impact analysis.
7. Glossary of Terms:
o Develop a Glossary of Terms for key terms and acronyms relevant to your project.Deliverables:
1. Written Report:
o Title Page: Include project title, course code (SE322), your name(s), and date.
o Table of Contents: A clear outline of the sections in the report.
o Project Overview: A brief introduction to the system, objectives, and scope.
o Use Case Descriptions: At least 5 detailed use cases using the provided template.
o Use-Case Diagram: Visual representation of actors and use cases.
o Functional and Non-Functional Requirements: A complete SRS document.
o Requirements Traceability Matrix (RTM): Ensuring traceability between use cases 
and requirements.
o Change Request Form: Documenting the change and its impact.
o Glossary of Terms: Definition of key terms and acronyms used in the project.
o Conclusion: Reflection on the project and lessons learned.
2. Presentation (10–15 minutes):
o Objective: Present the findings of your project to the class.
o Content:
▪ Brief overview of the project scenario.
▪ Explanation of key use cases and use-case diagram.
▪ Summary of functional and non-functional requirements.
▪ Discussion of the change request and how it impacted the system.
▪ Conclusion and any challenges encountered.
o Format: PowerPoint or Google Slides presentation or PDFs.
